# Contributing to Decision Security Engineering

Thank you for your interest in contributing to the Decision Security Engineering (DSE) project.

This repository contains the evolving specification, research paper, reference architecture, and reference implementation for Decision Security Engineering.

## Ways to Contribute

We welcome contributions including:

- Technical reviews
- Research discussions
- Literature recommendations
- Architecture feedback
- Reference implementations
- Benchmark datasets
- Documentation improvements
- Bug reports
- Security analysis

## Research Principles

Contributors are encouraged to:

- Prefer evidence over opinion.
- Clearly distinguish demonstrated results from proposed concepts.
- Cite authoritative sources where appropriate.
- Maintain implementation-independent architectural discussions whenever possible.

## Pull Requests

Please keep pull requests focused and well documented.

Large architectural changes should first be discussed through an RFC.

## Issues

When opening issues, please include:

- Problem description
- Expected behavior
- Suggested improvement
- References (if applicable)

## Code Style

Reference implementations should prioritize:

- Readability
- Reproducibility
- Explainability
- Deterministic behavior where practical

## Community

Decision Security Engineering is intended as an open engineering discussion.

Constructive technical disagreement is encouraged.

Respectful collaboration is expected from all participants.
