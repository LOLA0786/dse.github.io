# Security Policy

## Reporting Security Issues

If you discover a potential security vulnerability in the Decision Security Engineering reference implementation, please report it responsibly.

Please include:

- A clear description of the issue
- Steps to reproduce
- Potential impact
- Suggested remediation (if available)

Please do not publicly disclose vulnerabilities until they have been reviewed.

## Scope

This policy applies to:

- Reference implementation
- Runtime validation components
- Documentation
- Research artifacts
- Demonstration code

It does not apply to third-party software referenced by this repository.

## Research Disclaimer

The Decision Security Engineering framework is an evolving research effort.

Some components represent architectural concepts or reference implementations and should not be interpreted as production-certified security controls.

## Coordinated Disclosure

We support responsible and coordinated disclosure of security issues and will acknowledge valid reports.

Thank you for helping improve the security and quality of this project.
